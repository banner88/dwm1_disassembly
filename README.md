# Dragon Warrior Monsters (GBC) — Annotated Disassembly & Custom Game Editor

A work-in-progress annotated disassembly of Dragon Warrior Monsters (1998,
Game Boy Color, Enix/TOSE) that assembles **byte-identically** to the
original ROM, plus a proven patch system for custom rooms, NPCs, dialogue,
and story content. End goal: a visual editor for building custom stories.

**ROM**: Dragon Warrior Monsters (USA)
**Original MD5**: `1ca6579359f21d8e27b446f865bf6b83` ← the only canonical hash
**Assembler**: [RGBDS](https://github.com/gbdev/rgbds) v0.6.1 exactly

## Quick start

```bash
git clone https://github.com/banner88/dwm1_disassembly.git
cd dwm1_disassembly

# RGBDS v0.6.1 (all four tools)
git clone --branch v0.6.1 --depth 1 https://github.com/gbdev/rgbds.git /tmp/rgbds
sudo apt-get install -y bison
make -C /tmp/rgbds && sudo cp /tmp/rgbds/{rgbasm,rgblink,rgbfix,rgbgfx} /usr/local/bin/

# Provide the original ROM (gitignored; required by tools + verifier)
mkdir -p data && cp /path/to/DWM-original.gbc data/

# Verify everything (clean build byte-perfect + patched build + doc sanity)
python3 tools/verify_integrity.py
```

`verify_integrity.py` must PASS before and after any work session. If the
clean build's MD5 ever differs from `1ca657…`, the disassembly has drifted —
fix that first, and never "update" the expected hash.

## Building

```bash
# Clean (byte-perfect) ROM
cd disassembly
rm -f game.o game.gbc game.sym game.map   # NEVER `make clean` — see below
make                                       # → game.gbc, MD5 1ca657…

# Custom-content (patched) ROM
cp ../patches/*.asm . && make              # → modified game.gbc
git checkout -- .                          # restore clean tree afterwards
rm -f bank_060.asm bank_064.asm bank_067.asm bank_069.asm bank_06a.asm \
      bank_071.asm bank_072.asm bank_07e.asm   # the 8 new-bank patch files
```

> **NEVER run `make clean`.** It deletes committed `.2bpp` graphics that
> cannot be regenerated byte-identically. Remove only the four build
> artifacts listed above.

> **NEVER refactor `disassembly/`.** Labels and comments only. Anything
> that changes emitted bytes belongs in `patches/`. (This rule exists
> because a past refactor silently broke byte-perfection for several
> sessions — see PROJECT_STATE.md.)

## What works (proven in-game)

Custom rooms with multi-screen scrolling and exits, NPCs with scripted
branching dialogue (YES/NO), multi-page text, item give with inventory
checks, and event-flag story progression — all generated from
`patches/bank_060.asm` plus small same-size intercepts in banks
$00/$01/$04/$0B/$17. See `documentation/PROJECT_STATE.md` for the full
status dashboard and `documentation/CROSSBANK_ROOMS.md` for how it works.
